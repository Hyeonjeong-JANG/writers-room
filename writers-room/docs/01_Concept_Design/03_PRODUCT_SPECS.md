# Product Specs: Writer's Room
> Created: 2026-04-25 10:30
> Last Updated: 2026-04-25 10:30

## 1. 제품 개요

Writer's Room은 AI 에이전트 팀이 모여 드라마/웹소설의 다음 챕터를 토론하고 생성하는 온체인 공동 창작 플랫폼이다.

| 항목 | 값 |
|:---|:---|
| 제품 유형 | Responsive Web Application |
| 프레임워크 | Next.js (App Router) |
| 블록체인 | Base (Coinbase L2) |
| AI 추론 | FLock API (`api.flock.io/v1`, OpenAI SDK 호환) |
| 웹 데이터 수집 | Selanet (DePIN 기반 AI 웹 인프라) |
| 온체인 분석 | Nansen API (지갑 라벨링, Smart Money 분석, x402 pay-per-call) |
| 지갑 | Coinbase Smart Wallet (가스리스) |
| 결제 | x402 USDC (Base) |

## 2. 사이트맵

```
/                           -- 랜딩 페이지 (소개, CTA)
/stories                    -- 스토리 탐색 (진행중, 완결, 인기순)
/stories/[id]               -- 스토리 상세 (챕터 목록, 댓글)
/stories/[id]/chapter/[n]   -- 챕터 읽기
/stories/[id]/room          -- 작가방 (에이전트 토론 뷰)
/stories/create             -- 새 스토리 생성
/agents                     -- 에이전트 마켓플레이스
/agents/[id]                -- 에이전트 프로필 (평점, 고용 이력, 비용)
/agents/create              -- 에이전트 생성/훈련
/dashboard                  -- 내 스토리, 내 에이전트, 수익 현황
/dashboard/earnings         -- x402 수익 내역 (온체인 기록 링크)
/profile/[address]          -- 유저 프로필 (기여 히스토리)
```

## 3. MVP 핵심 기능

### F1. 스토리 생성 및 관리
- 제목, 장르, 시놉시스, 초기 설정(세계관, 캐릭터) 입력
- 장르 선택: 로맨스, 판타지, 스릴러, SF, 드라마 등
- 스토리 상태 관리: 연재중 / 휴재 / 완결

### F2. AI 작가방 (Writer's Room)
- **에이전트 구성**: 스토리당 최소 3개 에이전트 배치 (PD, 작가, 편집자)
  - PD: 전체 방향성, 독자 반응 분석, 의사결정
  - 작가: 실제 텍스트 생성, 캐릭터 대사, 장면 묘사
  - 편집자: 일관성 검토, 문법/스타일 교정, 품질 관리
- **토론 프로세스**:
  1. 이전 챕터 컨텍스트 + 채택된 독자 댓글을 입력으로 받음
  2. 백그라운드에서 에이전트 간 멀티턴 토론 실행 (FLock API)
  3. 토론 완료 후 요약본을 유저에게 표시
  4. 유저가 원하면 전체 토론 로그 확인 가능
  5. 토론 결과를 바탕으로 다음 챕터 초안 생성
  6. 유저(창작자)가 검토/수정 후 발행
- **기술 구현**:
  - FLock API 호출 (chat/completions endpoint)
  - 에이전트별 시스템 프롬프트로 역할 분리
  - 이전 챕터 + 세계관 + 캐릭터 설정을 컨텍스트로 관리

### F3. 커뮤니티 댓글 및 아이디어 선별
- 챕터별 댓글 섹션
- AI가 댓글 중 스토리에 활용 가능한 아이디어를 자동 선별
- 선별 기준: 창의성, 캐릭터/세계관 일관성, 커뮤니티 반응(좋아요 수)
- 채택된 댓글은 "이 아이디어가 다음 챕터에 반영되었습니다" 표시
- 채택 기여도 온체인 기록

### F4. 에이전트 마켓플레이스
- **에이전트 생성**: 유저가 장르/스타일 특화 에이전트 생성
  - 역할 선택 (PD / 작가 / 편집자)
  - 장르 전문성 설정 (로맨스, 판타지, 스릴러 등)
  - 커스텀 시스템 프롬프트 작성
  - 샘플 작품으로 에이전트 "훈련" (few-shot examples)
- **에이전트 고용**: 다른 유저가 x402 USDC로 에이전트를 고용
  - 고용 시 x402 프로토콜로 USDC 자동 결제
  - 플랫폼 수수료 (10%) 차감 후 빌더에게 지급
- **에이전트 프로필**: 평점, 고용 횟수, 참여 작품, 리뷰

### F5. x402 USDC 결제
- Coinbase Smart Wallet 연결 (소셜 로그인으로 자동 생성)
- 에이전트 고용 시 x402 프로토콜로 마이크로 페이먼트
- 결제 내역 온체인 기록 (Base)
- 수익 대시보드: 빌더의 에이전트별 수익 현황

### F6. 온체인 기여 및 평판 시스템
- **기여 기록**: 댓글 채택, 에이전트 참여, 챕터 생성 등 모든 기여를 Base에 기록
- **평판 점수**: 에이전트별 별점 평가 (1-5), 고용 횟수, 성공적 챕터 기여율
- **투명성**: 모든 거래/기여 내역이 Base Explorer에서 조회 가능

### F7. 트렌드 인텔리전스 (Selanet 연동)
- **실시간 장르 트렌드 수집**: Selanet의 DePIN 노드가 웹소설 플랫폼(네이버시리즈, 카카오페이지, Wattpad 등)에서 인기 장르/키워드/트렌드를 수집
- **PD 에이전트 컨텍스트 강화**: 토론 시작 시 Selanet에서 수집한 최신 트렌드 데이터를 PD 에이전트의 컨텍스트에 주입
- **문화 이슈 반영**: 뉴스, SNS, 커뮤니티에서 실시간 이슈를 수집해 스토리 소재로 제안
- **경쟁 작품 패턴 분석**: 비슷한 장르의 인기 작품 구조를 분석해 편집자 에이전트 피드백 품질 향상

### F8. Nansen Trust Score (에이전트 신뢰 평가)
- **온체인 행동 기반 신뢰 점수**: Nansen API(x402 pay-per-call)로 에이전트 관련 온체인 데이터를 분석해 신뢰 점수 산출
- **신뢰 지표 4가지**:
  - 재고용률: 동일 지갑이 같은 에이전트를 반복 고용한 비율
  - 고품질 고용자 비율: Smart Money/Verified 지갑의 고용 비율
  - 활용률: 에이전트 고용 후 실제 챕터가 발행된 비율
  - 도달 범위: 에이전트를 고용한 고유 지갑 수
- **Nansen Trust Badge**: 신뢰 점수 기준으로 에이전트 프로필에 뱃지 부여 (Bronze/Silver/Gold)
- **유저 지갑 검증**: 고용자의 지갑을 Nansen으로 조회해 Smart Money/고래 여부 표시
- **사기 방지**: 신규 지갑의 비정상 대량 결제 시 Nansen 리스크 체크

## 4. 사용자 플로우

### Flow A: 창작자가 스토리를 만들고 챕터를 생성하는 흐름

```
가입 (Smart Wallet)
  -> 새 스토리 생성 (제목, 장르, 시놉시스, 캐릭터 설정)
  -> 에이전트 배치 (기본 3명 또는 마켓에서 고용)
  -> [에이전트 고용 시] x402 USDC 결제
  -> 작가방 열기
  -> "다음 챕터 토론 시작" 클릭
  -> 백그라운드 토론 진행 (로딩/진행률 표시)
  -> 토론 요약 확인 (원하면 전체 로그)
  -> 챕터 초안 확인
  -> 수정/편집
  -> 발행
```

### Flow B: 독자가 댓글로 기여하는 흐름

```
스토리 탐색
  -> 챕터 읽기
  -> 댓글 작성 (아이디어 제안)
  -> [AI 선별] 아이디어가 작가방 토론에 투입됨
  -> 다음 챕터에 반영 시 알림 수신
  -> 기여도 온체인 기록
  -> 프로필에서 기여 히스토리 확인
```

### Flow C: 에이전트 빌더의 수익화 흐름

```
에이전트 생성 (역할, 장르, 프롬프트, 샘플 작품)
  -> 마켓플레이스에 등록
  -> 다른 유저가 고용
  -> x402 USDC 자동 수령 (수수료 차감 후)
  -> 대시보드에서 수익/평판 확인
  -> 리뷰 기반으로 에이전트 개선
```

## 5. Non-Functional Requirements (NFR)

| 항목 | 요구사항 |
|:---|:---|
| **응답 시간** | 에이전트 토론 시작 -> 요약 표시: 30초 이내 (프로그레스 바 표시) |
| **챕터 생성** | 토론 결과 -> 챕터 초안: 15초 이내 |
| **결제 확인** | x402 USDC 결제 확정: 5초 이내 (Base L2) |
| **동시 접속** | MVP 기준 100명 동시 접속 지원 |
| **데이터 보존** | 모든 스토리/챕터는 DB에 영구 보존, 기여 기록은 온체인 |
| **보안** | Smart Wallet 기반 인증, API 키 서버사이드 관리, Rate Limiting |
| **접근성** | 모바일 반응형 필수, 한국어/영어 2개 언어 지원 |

## 6. 기술 스택 (초안)

| 레이어 | 기술 | 선택 근거 |
|:---|:---|:---|
| Frontend | Next.js 15 (App Router) + TypeScript | SSR/RSC 지원, 풀스택 가능 |
| Styling | Tailwind CSS + shadcn/ui | 빠른 프로토타이핑, 반응형 |
| 상태 관리 | Zustand 또는 React Server Components | 경량, 서버 컴포넌트 활용 |
| DB | Supabase (PostgreSQL) | 실시간 구독, Auth 내장, 빠른 개발 |
| AI 추론 | FLock API (`api.flock.io/v1`) | OpenAI SDK 호환, 저비용 |
| 웹 데이터 | Selanet API | DePIN 기반 웹 자동화, AI 에이전트용 트렌드 수집 |
| 온체인 분석 | Nansen API (x402) | 5억+ 지갑 라벨링, Smart Money 분석, pay-per-call |
| 블록체인 | Base (Coinbase L2) | 저가스비, Coinbase 생태계 |
| 지갑 | Coinbase Smart Wallet (OnchainKit) | 가스리스, 소셜 로그인 |
| 결제 | x402 프로토콜 (USDC on Base) | 마이크로 페이먼트 최적화 |
| 배포 | Vercel | Next.js 최적 호스팅 |

## 7. 제약 사항 및 의존성

| 제약 | 영향 | 대응 |
|:---|:---|:---|
| FLock API 모델 가용성 | 사용 가능한 모델에 따라 에이전트 품질 결정 | 모델 목록 API로 동적 대응, 폴백 모델 설정 |
| x402 프로토콜 성숙도 | 아직 초기 단계, SDK/문서 변동 가능 | Coinbase 공식 문서 추적, 폴백으로 직접 USDC transfer |
| Base 가스비 변동 | 트랜잭션 비용 예측 어려움 | Smart Wallet의 가스 스폰서링 활용 |
| 컨텍스트 윈도우 한계 | 긴 스토리의 전체 맥락 유지 어려움 | 요약 + RAG 패턴으로 컨텍스트 관리 |
